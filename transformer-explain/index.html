Input Embedding

Token Representation / 词元嵌入

Embedding 层将离散的 token ID 映射为连续的向量表示。在 Transformer 中，每个输入 token 都被转换为一个 d_model 维的稠密向量。

传统的 Word2Vec、GloVe 等预训练词向量可以被直接加载使用，也可以在任务中随机初始化并端到端训练。Embedding 矩阵的形状为 [vocab_size, d_model]。

每个 token 经过 Embedding 层后，获得一个固定维度的语义表示向量，捕获了该 token 的语法和语义信息。

数据维度
输入 → [B, L]
输出 → [B, L, D]
关键参数
vocab_size = ~50k
d_model = 512
参数量 = vocab × D
📍

Positional Encoding

位置编码 / 序列顺序

Transformer 不包含循环结构，因此无法自然捕获序列中 token 的位置信息。位置编码通过向 Embedding 添加位置信号来解决这个问题。

原论文使用正弦和余弦函数交替编码位置：

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

这种编码方式允许模型学习相对位置关系（因为 sin(A+B) 和 cos(A+B) 可以表示 A 和 B 的关系）。

数据维度
输入 → [B, L, D]
输出 → [B, L, D]
设计动机
可扩展 → 处理任意长度
确定性 → 无需学习
相对位置 → 天然支持
🔍

Multi-Head Self-Attention

多头自注意力机制

Self-Attention 是 Transformer 的核心创新。它允许序列中的每个位置关注序列中的所有其他位置，捕获长距离依赖关系。

对于输入 X，我们学习三个线性投影：查询 Q、键 K、值 V。注意力输出为：

Attention(Q,K,V) = softmax(QK^T / √d_k) × V

Multi-Head 将注意力分散到多个"头"中，每个头学习不同的注意力模式（有些头关注语法，有些关注语义等）。

MultiHead = Concat(head_1, ..., head_h) × W_O

数据维度
输入 X → [B, L, D]
Q, K, V → [B, L, D]
注意力分数 → [B, h, L, L]
输出 → [B, L, D]
关键参数
h (heads) = 8
d_k = d_v = D/h = 64
🎭

Masked Self-Attention

因果遮蔽注意力 / Decoder 自回归

Masked Self-Attention 确保 Decoder 在预测第 t 个 token 时，只能看到位置 1 到 t-1 的信息（不能看到当前及未来的 token）。

这通过在计算注意力分数后、应用 softmax 之前，将不该看到的位置设置为 -∞ 或非常大的负数来实现。

softmax(-∞) = 0，所以这些位置的注意力权重变为 0，实现了"因果"（causal）或"自回归"（autoregressive）特性。

这种设计是 GPT、LLaMA 等自回归语言模型的核心，它们在生成每个 token 时只能基于之前生成的 token。

数据维度
输入 → [B, L, D]
Mask 形状 → [L, L]
Mask 效果 → 上三角 → -∞
输出 → [B, L, D]
核心特点
并行化 → 训练时全序列并行
推理 → 逐 token 自回归
🔗

Cross Attention

编码器-解码器注意力 / 跨模态交互

Cross Attention 连接 Encoder 和 Decoder，让 Decoder 的每个位置可以关注 Encoder 的所有位置。这是序列到序列转换（如翻译）的关键机制。

与 Self-Attention 的区别：

• Q 来自 Decoder 的前一层的输出
• K 和 V 来自 Encoder 的最终输出

这使得每个 Decoder 位置都能"查询"整个源序列的信息。例如，在翻译任务中，Decoder 的每个位置可以关注源语言句子的相关部分。

数据维度
Q ( Decoder ) → [B, L_dec, D]
K, V ( Encoder ) → [B, L_enc, D]
注意力矩阵 → [B, L_dec, L_enc]
输出 → [B, L_dec, D]
应用场景
机器翻译 → Q: 目标语言
Vision Transformer → Q: 文本, KV: 图像
⚡

Feed Forward Network

前馈神经网络 / 位置智能变换

FFN 是每个 Transformer 层中的两个线性变换，中间有一个 ReLU 激活：

FFN(x) = max(0, xW₁ + b₁)W₂ + b₂

尽管名字叫"前馈网络"，但它实际上是逐位置（point-wise）应用的——即对序列中每个位置的向量独立应用相同的变换。

FFN 的作用是引入非线性变换，让每个位置可以学习更复杂的表示。它通常将维度从 d_model 扩展到 d_ff（通常是 4 倍），再压缩回来。

Attention 负责"信息流动"（哪个位置该关注哪些），FFN 负责"信息变换"（每个位置应该如何表示）。

数据维度
输入 → [B, L, D]
隐藏层 → [B, L, d_ff]
输出 → [B, L, D]
关键参数
d_model = 512
d_ff = 2048 (4×D)
参数量 = 2 × D × d_ff
📊

Layer Normalization

层归一化 / 梯度稳定

LayerNorm 对每一层的输入进行归一化，使其均值为 0、方差为 1，然后通过可学习的缩放和偏移参数调整：

LN(x) = γ ⊙ (x - μ) / √(σ² + ε) + β

与 BatchNorm 不同，LayerNorm 在特征维度上归一化，对每个样本独立处理。这使得它对序列长度变化更鲁棒，适合变长序列。

Add & Norm 结构包含一个残差连接（x + Sublayer(x)）加上 LayerNorm。残差连接确保梯度能够直接回传，缓解深层网络的梯度消失问题。

数据维度
输入 → [B, L, D]
归一化维度 → D (最后一维)
输出 → [B, L, D]
关键参数
γ (scale) = D
β (shift) = D
ε = 1e-6 (数值稳定)

维度符号速查表

B
Batch Size
一个 batch 中的样本数量，通常为 16-64
L
Sequence Length
输入/输出序列的 token 数量，通常为 128-4096
D
d_model
模型维度，Token 嵌入和隐藏层的向量维度，常见为 512/768/1024
d_k
Key Dimension
每个注意力头的 key 向量维度 = D / h
d_v
Value Dimension
每个注意力头的 value 向量维度 = D / h
d_ff
Feed Forward Dim
FFN 隐藏层维度，通常为 d_model 的 4 倍 (2048/3072/4096)
h
Num Heads
注意力头数量，允许模型同时学习多种注意力模式
V
Vocabulary Size
词表大小，GPT-3 为 50257，中文通常 50000+
